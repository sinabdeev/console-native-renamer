## 🔍 Описание

**File Renamer** — это консольная утилита на **Kotlin/Native**, которая переименовывает файлы в указанной папке, присваивая им случайные имена в формате:

```
<цифра>-<цвет>-<птица>.<расширение>
```

**Пример:**
```
📄 IMG_20240315.jpg  →  7-blue-eagle.jpg
📄 report_final.pdf  →  3-red-sparrow.pdf
```

Программа не требует установки Java или Kotlin — это **нативный исполняемый файл**, готовый к запуску.

## ✨ Возможности

| Функция | Описание |
|---------|----------|
| 🎲 Случайные имена | Генерация уникальных имён из комбинации цифры, цвета и названия птицы |
| 📁 Работа с путями | Чтение пути к папке из конфигурационного файла |

| ⚠️ Обработка ошибок | Программа продолжает работу даже при ошибке с одним файлом |
| 🚀 Нативная компиляция | Быстрый запуск без зависимостей (JVM не требуется) |

## 🚀 Быстрый старт

### 1️⃣ Скачайте готовый исполняемый файл

Найдите `MyNativeApp.exe` в папке:
```
build/bin/mingwX64/ReleaseExecutable/
```

### 2️⃣ Создайте файл конфигурации `config.ini`

В той же папке, где лежит `.exe`, создайте файл `config.ini`:

```ini
folder_path=C:/Users/user/Pictures/ToRename
```

> 💡 Используйте прямые слеши `/` или двойные обратные `\\` в путях Windows.

### 3️⃣ Подготовьте файлы для переименования

Убедитесь, что в указанной папке есть файлы:
```
C:/Users/user/Pictures/ToRename/
├── photo1.jpg
├── document.pdf
└── video.mp4
```

### 4️⃣ Запустите программу

Дважды кликните по `MyNativeApp.exe` **или** запустите через консоль:

```cmd
MyNativeApp.exe
```

### 5️⃣ Проверьте результат

```
File Renamer started...
Found 3 files to rename
Renamed: photo1.jpg -> 5-blue-sparrow.jpg
Renamed: document.pdf -> 2-red-eagle.pdf
Renamed: video.mp4 -> 8-green-owl.mp4
Renaming completed. 3 files renamed.
```

### Файл `config.ini`

| Параметр | Обязательный | Описание | Пример |
|----------|--------------|----------|--------|
| `folder_path` | ✅ Да | Путь к папке с файлами для переименования | `C:/MyFiles` |

**Пример полного конфига:**
```ini
# Конфигурация File Renamer

```

> 📝 Программа ищет `config.ini` в **текущей рабочей директории** (откуда запущена).

## 🎲 Формат имён файлов

Имя генерируется по шаблону:  
**`<ЦИФРА>-<ЦВЕТ>-<ПТИЦА>`**

### Доступные значения:

| Компонент | Варианты | Примеры |
|-----------|----------|---------|
| **Цифра** | `0` – `9` | `3`, `7`, `0` |
| **Цвет** | `red`, `blue`, `green`, `yellow`, `purple`, `orange`, `pink`, `brown`, `black`, `white` | `blue`, `purple` |
| **Птица** | `eagle`, `owl`, `sparrow`, `robin`, `penguin`, `parrot`, `hawk`, `swan`, `peacock`, `flamingo` | `eagle`, `sparrow` |

### Примеры сгенерированных имён:

```
7-blue-eagle.jpg
3-purple-swan.pdf
0-green-hawk.mp4
9-red-owl.txt
```

> ⚠️ **Внимание:** Возможны коллизии! Два файла могут получить одинаковое имя. См. [Предупреждения](#-предупреждения).

## 🔨 Сборка из исходников

### 1️⃣ Клонируйте репозиторий

```bash
git clone https://github.com/yourusername/file-renamer.git
cd file-renamer
```

### 2️⃣ Откройте проект в OpenIDE / IntelliJ IDEA

*   `File` → `Open` → выберите папку проекта
*   Дождитесь импорта Gradle
*   Нажмите **🐘 Sync Now**, если появится уведомление

### 3️⃣ Соберите релизную версию

#### Через панель Gradle:
```
Gradle → Tasks → link → linkReleaseExecutableMingwX64
```

#### Или через терминал:
```bash
# Windows (PowerShell / CMD)
./gradlew linkReleaseExecutableMingwX64

# Linux / macOS
./gradlew linkReleaseExecutableLinuxX64
# или
./gradlew linkReleaseExecutableMacosArm64
```

### 4️⃣ Найдите исполняемый файл

```
build/bin/mingwX64/ReleaseExecutable/MyNativeApp.exe
```

## 🌍 Использование в других ОС

Проект настроен под **Windows (mingwX64)**, но легко адаптируется:

### Для Linux:
1.  В `build.gradle.kts` замените или добавьте:
    ```kotlin
    kotlin {
        linuxX64()  // или linuxArm64()
        // ...
    }
    ```
2.  Измените путь к исходникам: `src/linuxX64Main/kotlin/`
3.  Соберите: `./gradlew linkReleaseExecutableLinuxX64`

### Для macOS:
```kotlin
kotlin {
    macosX64()    // Intel
    macosArm64()  // Apple Silicon
}
```
> ⚠️ Для сборки под macOS требуется компьютер с macOS и установленный Xcode.

### Кроссплатформенная сборка (общий код):
```kotlin
kotlin {
    mingwX64()
    linuxX64()
    macosArm64()
    
    sourceSets {
        val commonMain by getting {
            // Общий код для всех платформ
        }
        val nativeMain by getting {
            dependsOn(commonMain)
            // Код, специфичный для Native (но общий для всех ОС)
        }
    }
}
```

## ⚠️ Предупреждения

| Риск | Описание | Как избежать |
|------|----------|--------------|
| 🔁 **Необратимость** | Переименованные файлы нельзя автоматически вернуть к исходным именам | ✅ Сначала протестируйте на копии папки |
| ⚔️ **Коллизия имён** | Два файла могут получить одинаковое случайное имя | ✅ Добавьте проверку `if (newFilePath.exists())` или увеличьте энтропию имени |
| 📁 **Только верхний уровень** | Программа не обрабатывает файлы в подпапках | ✅ Используйте `folder.walk()` для рекурсивного обхода |
| 🔐 **Права доступа** | Ошибка при переименовании, если нет прав на запись | ✅ Запускайте от имени администратора или выберите папку с правами |
| 🌐 **Кодировка путей** | Проблемы с кириллицей в путях на некоторых системах | ✅ Используйте ASCII-пути или протестируйте заранее |
